# Landing page

This is the landing page that TruckTel serves on (by default) port 8080. It
gets bundled into the plugin by means of a header file, generated by
`bundle.py`.

The git repository contains a placeholder header file, so you don't strictly
need to generate it when you're developing locally. You can regenerate the
placeholder with `./bundle.py placeholder`. CI will always replace this with
the real one when building a release.

You can of course build the real one yourself as well. Just run `./bundle.py`,
without the `placeholder` argument. You need Python and NPM installed for this.

You can also test the landing page with the development server using
`npm run dev`. A default info object will be served by the development server
in that case; see public/api/info.json. Modify it if you want to test changes
to the landing page.

The landing page is intentionally quite lean, e.g. I tore out MDI and just used
emoji because of how large the fonts are. The larger the rendered page becomes,
the longer it'll take the compiler to parse the header file.
